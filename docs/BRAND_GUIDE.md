# NPVT Terminal Converter — Brand Guide

## Visual Direction

The project identity is built around a polished green terminal interface:

- Dark cyber-terminal background
- Neon green glow
- Clean code-editor panels
- Glass-like cards
- Minimal protocol chips
- Small `TheLouisMahdi` signature

## Color Palette

| Token | Hex | Usage |
|---|---:|---|
| Terminal black | `#020704` | Main background |
| Deep green | `#06140b` | Panels and shadows |
| Neon green | `#39ff88` | Main accent |
| Signal green | `#00d46a` | Secondary accent |
| Soft mint | `#b8ffd2` | Highlights and text |
| Muted green | `#82ad92` | Secondary text |
| Warning amber | `#ffcb6b` | Warning buttons |
| Error pink | `#ff6688` | Error dots and alerts |

## Typography

Recommended feel:

- UI text: clean sans-serif
- Technical labels: monospace
- Buttons and chips: uppercase monospace

## Repository Presentation Rules

- Keep the README visually rich but still readable.
- Use badges only for real project traits.
- Keep the main identity green and black.
- Avoid fake official logos.
- Avoid aggressive hacker visuals.
- Keep the project looking like a legitimate developer utility.
